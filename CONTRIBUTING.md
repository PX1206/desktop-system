# 参与贡献

感谢你对「云同步」项目的关注。欢迎通过 Issue、Pull Request 参与改进。

## 如何贡献

1. **Fork** 本仓库，在分支上修改（建议命名：`fix/…`、`feat/…`）。
2. 保持变更**聚焦**：一次 PR 解决一类问题，避免无关大改。
3. 提交信息使用清晰的中文或英文说明「做了什么、为什么」。
4. 发起 **Pull Request**，描述变更范围与验证方式（如何本地启动、如何自测）。

## 请勿提交敏感信息

- **不要**将生产环境数据库密码、Redis 密码、JWT/密钥、阿里云短信 Key、RSA 私钥等写入仓库。
- 本地 `application.yml`、`application-prod.yml`、`.env` 若含真实密钥，请使用**示例文件**（如 `application-example.yml`）或 `.gitignore` 排除；PR 前检查 diff。
- 若误提交密钥，请立即轮换线上凭证，并考虑使用 `git filter-repo` 等工具清理历史（需仓库管理员配合）。

## Issue 范围

- **Bug**：复现步骤、期望行为、实际行为、环境（OS、JDK/Node 版本、分支）。
- **功能建议**：使用场景与优先级说明。
- **文档**：指出文件路径与建议修改方向。

## 代码风格（约定）

- **Java**：与现有模块保持一致（包名、Lombok、MyBatis-Plus 用法）。
- **Vue/TypeScript**：与现有组件风格一致；避免无必要的格式化全文件变更。

## 许可证

向本仓库贡献的代码，默认视为在 [LICENSE](./LICENSE)（MIT）下授权，与项目整体保持一致。
